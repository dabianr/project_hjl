# DEBUG日志 — 2026-05-15

## 问题1: /admin/dashboard SQL 语法错误

**现象**: 后端无法启动，SyntaxError
**根因**: `sed -i` 插入 SQL 用双引号包裹，SQL 内 `DATE("now", "-6 days")` 的内引号提前关闭了 Python 字符串
**修复**: 改为 `DATE('now', '-6 days')`（单引号）

## 问题2: device_id 不继承旧记录

**现象**: 你的存证数显示 0，旧记录 16 条不显示
**根因（v1）**: device_id 查 DB 时 `WHERE uploader = device_id` 无匹配（旧记录 uploader="anonymous"）
**修复（v1）**: 加 `elif not Web3.is_address`: fallback 到链上总数
**残留**: 一旦 device_id 上传过新文件，DB 有匹配就走到 `row[0]` 分支，只显示新记录数
**修复（v2）**: device_id 不走 DB 筛选，直接返回链上总数

## 问题3: useEffect 依赖缺失

**现象**: 从门户页进入主界面后 fetchStats 不带 device_id
**根因**: `useEffect` 依赖 `[account]`，`hasDeviceId` 变化不触发重跑
**修复**: 依赖改为 `[account, hasDeviceId]`

## 问题4: 管理员控制台设备数虚高

**现象**: 2 台设备显示 5 台
**根因**: 清缓存 → localStorage 清空 → 进门户生成新 UUID → 每个 UUID 算独立设备
**修复（v1）**: 非地址 UUID 合并为「网页用户」
**回退**: 用户接受 UUID 变化，要求按密钥独立显示
**修复（v2）**: `most_common(10)` + 新增 `total_device_count` 全量去重数

## 问题5: 使用率一直为 0%

**现象**: RingChart 始终 0%
**根因**: 分母写死 10000，18/10000×100=0.18% → Math.round → 0
**修复**: 分母改为合约 MAX_RECENT=200，18/200×100=9%

## 问题6: 门户页登录需手动刷新

**现象**: 进门户后数据不更新，需手动 F5
**修复**: `onEnter={() => setHasDeviceId(true)}` 改为 `window.location.reload()`

## 问题1: Top 5 显示多于 5 条

**现象**: 设备 Top 5 显示了 10 条
**根因**: 回退时 `most_common` 参数从 5 改成了 10

## 问题2: 使用率一直 0%

**现象**: RingChart 始终 0%
**根因**: 分母 10000，18/10000×100=0.18% → `Math.round` → 0
**修复**: 分母改为合约 `MAX_RECENT=200`，18/200×100=9%

## 问题3: 设备总数从 9 变成 5

**现象**: MiniStat 设备数显示 5
**根因**: 取 `device_stats` 的 key 数（仅 Top 5），非全量
**修复**: 后端新增 `total_device_count` 字段，前端改用该字段

## 问题4: 换密钥登录看到同样存证数

**现象**: 不同密钥显示的 your_evidence_count 相同
**根因**: 非地址 uploader 走了 fallback 到链上总数
**修复（v1）**: 改为无钱包时直接显示总数
**回溯**: 用户澄清 device_id 应作为临时账户
**修复（v2）**: 统一走 DB 按 uploader 精确统计，无匹配则 0

## 问题5: 门户页登录后数据不更新

**现象**: 进门后需手动 F5
**根因**: `onEnter` 只改了 `hasDeviceId` 状态，useEffect 依赖复杂
**修复**: `onEnter` 改为 `window.location.reload()`，进门全刷新
