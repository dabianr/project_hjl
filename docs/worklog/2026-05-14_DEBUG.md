# DEBUG日志 — 2026-05-14（下午）

## 问题1: useEffect 内 return JSX 导致编译报错

**现象**: 前端 npm start 报编译错误
**根因**: `App.js` 的 `useEffect` 回调里写了 `return <PortalPage .../>`，React useEffect 的 cleanup 只能返回函数或 undefined
**修复**: 删除 useEffect 内的 JSX return，将门户页渲染逻辑放到组件主体（`if (!hasDeviceId) return <PortalPage />`）

## 问题2: 沙箱写入文件权限导致前端无法读取

**现象**: `npm run build` 报 EACCES
**根因**: 沙箱`write_file`/`sudo cp` 写入的文件属主 root，权限 640，npm 进程读不到
**修复**: `sudo chmod -R o+rX frontend/src`，package.json 等项目文件也要 `chmod o+r`

## 问题3: 后端 /admin/login 返回 404

**现象**: `POST /admin/login` 返回 `{"detail":"Not Found"}`
**根因**: 后端没重启，旧版本 main.py 没有 JWT 端点
**修复**: 需要用户手动 `fuser -k 8001/tcp` 后重启 uvicorn

## 问题4: 后端 /stats 抛异常（device_id 传入 blockchain_service）

**现象**: `get_contract_stats(device_id)` 抛 InvalidAddress
**根因**: `w3.to_checksum_address(uploader_address)` 对 UUID 字符串报错
**修复(v1)**: main.py 对 stats 加 try/except — 太暴力，链上总数也被吞
**修复(v2)**: blockchain_service.py 加 `Web3.is_address(uploader_address)` 判断，非地址跳过链上查询
**结果**: 优雅，不影响 total_evidence_count

## 问题5: device_id 用户 your_evidence_count 为 0

**现象**: 大门户后用 device_id 登录，你的存证数显示 0
**根因（前次修复）**: 上传时传了设备 UUID → DB 存 uploader=device_id → stats 查 DB 时 `WHERE uploader = device_id` 无匹配（旧记录 uploader="anonymous"）
**修复（前次）**: `elif not Web3.is_address(uploader): stats["your_evidence_count"] = stats["total_evidence_count"]`
**残留问题**: 一旦 device_id 用户上传过新文件，DB 有匹配记录就走不到 fallback，只显示新记录数
**最终修复**: device_id 不走 DB 筛选，直接返回链上总数

## 问题6: 管理员控制台设备数虚高

**现象**: 用户只用 2 台设备但 dashboard 显示 5 台
**根因**: 清缓存 → localStorage device_id 丢失 → 进门户生成新 UUID → 每个 UUID 在 DB 算独立设备
**修复**: wallet 地址独立统计，非地址 UUID 全部合并为「网页用户」

## 问题7: useEffect 依赖缺失导致 fetchStats 不传 device_id

**现象**: 从门户页进入主界面后，你的存证数始终不更新
**根因**: App.js `useEffect(() => { ... fetchStats(id) }, [account])` 依赖只有 `[account]`。用户进门户 → `setHasDeviceId(true)` → 重渲染但 account 没变 → effect 不重跑 → `fetchStats(deviceId)` 从未执行
**修复**: 依赖改为 `[account, hasDeviceId]`

## 问题8: 前端密钥无法复制

**现象**: 点击 PortalPage 复制按钮无反应
**根因**: `navigator.clipboard.writeText()` 在非 HTTPS 下静默失败，空的 `catch {}` 吃掉异常
**修复**: 加 fallback — 创建临时 textarea → select → `document.execCommand("copy")` → 移除

## 问题9: /admin/dashboard SQL 引号语法错误

**现象**: 后端无法启动，`SyntaxError: invalid syntax`
**根因**: `sed -i` 插入 SQL 时用了双引号包裹，SQL 内 `DATE("now", "-6 days")` 的内引号提前关闭了 Python 字符串
**修复**: 改为 `DATE('now', '-6 days')`（单引号）
