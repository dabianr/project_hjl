# DEBUG日志 — 2026-05-12

## 问题1: 复制按钮不工作

**现象**: 点击复制按钮无反应（EvidenceList / VerifyTool）
**根因**: `navigator.clipboard.writeText()` 需要安全上下文（HTTPS），通过自签证书 nginx 反代时可能被浏览器拒绝
**修复**: 添加 fallback 方案 `document.execCommand("copy")`，使用临时 textarea 元素 + select + execCommand
**验证**: 浏览器手动测试复制哈希/IPFS/交易哈希

## 问题2: 大文件上传卡在90%且超时

**现象**: 远程上传 >2MB 文件时进度条到 90% 不动，最终超时
**根因**: 
  - 进度条是前端 `setInterval` 模拟的（非真实上传进度），到 90% 后停止等待后端响应
  - nginx `client_max_body_size` 默认 1MB（已修）
  - Cloudflare tunnel 可能在大文件上传时有缓冲/超时
**沙箱验证**: 3MB 文件直接走 nginx 本地上传 46ms 完成，后端无性能问题
**修复**:
  1. `nginx.nix` 加 `client_max_body_size 50M;` + `proxy_request_buffering off;`
  2. UploadDropzone.jsx: 模拟进度条改为 axios `onUploadProgress` 真实进度
  3. 添加 axios `timeout: 120000` 防止无限等待
**待验证**: 用户需 `sudo nixos-rebuild switch` 后远程测试

## 问题3: 文件上传后消失（已修复于持久化修复）

**修复**: start.sh 加 `anvil --state .anvil/ --state-interval 30`
**状态**: ✅ 已修复

## 问题4: 公网 API 404（已修复）

**现象**: `/api/stats` 返回 `{"detail":"Not Found"}`
**根因**: nginx `proxyPass` 无尾斜杠 + NixOS 模块可能不生成某些 location
**修复**: `locations."/api"` 加 `rewrite ^/api/?(.*)$ /$1 break;`
**状态**: ✅ 已修复

## 问题5: 公网后端端点 404 纯 GET（疑云）

**现象**: `/api` (无斜杠) 返回后端根服务信息，但 `/api/` + `/api/stats` 返回 404
**根因**: nginx location `/api` 匹配时，`proxyPass` 路径行为：不带尾斜杠则原样转发 URI
**修复**: rewrite 剥离 `/api` 前缀
**状态**: ✅ 已修复
