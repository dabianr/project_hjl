# 2026-05-14 工作日志（也许是最终版）

项目：BlockProof — 区块链电子文件存证系统

## 核心改动

### 前端
- **EvidenceList 分页重写**：从跨组件传参改为本地 `useState` 自管理，解决翻页回弹问题
- **分页 total 获取**：组件挂载时独立调用 API 获取总记录数，翻页控件正确显示
- **暗色卡片修复**：`dark` class 从 `<div>` 移到 `<html>`（`document.documentElement.classList.toggle`），`evidence-card` 类选择器正确匹配
- **证据卡片 UI**：暗色 `#1a1a2e`（深紫蓝），日间 `#f0f0f5`（浅灰白），与页面底色明显区分
- **生产 build**：`npm run build` 产出带 hash 文件名，nginx 托管 `/var/www/blockproof/`，Cloudflare 无法缓存旧版
- **`.env.development`**：`REACT_APP_API_URL=http://127.0.0.1:8001`，本地开发直连后端

### 后端
- **`/logs` total 修复**：返回真实总数而非当前页数量（`SELECT COUNT(*)`）

### 基础设施
- **nginx**：多次尝试 `root + tryFiles` 静态托管均 500，最终回退 proxy 模式
- **NixOS nginx 模块**：`root` + `tryFiles` 在模块中始终报 500，绕过方案改用 `extraConfig` raw nginx 语法

## 经验教训

1. `dark` class 必须挂在 `<html>` 上，CSS 选择器 `html:not(.dark)` 才能生效
2. 分页状态用组件本地 `useState` 管，不要跨组件传参
3. 每次修改后必须 `npm run build` 验证，不能只靠 HMR
4. 删除变量后要全局搜索是否还有残留引用
5. NixOS nginx 模块的 `root` + `tryFiles` 会报 500
6. commit message 必须简短精炼
